# Sprint Planning & Milestones — SST

## Purpose

Sequence delivery into sprints and milestones.

## Audience

Engineering lead, developers.

## Scope

MVP (~8 sprints @ 1–2 weeks assuming solo/pair). Adjust capacity freely.

## Definitions

| Milestone | Outcome |
|-----------|---------|
| M1 | Foundations + requirements |
| M2 | Candidates |
| M3 | Offers + onboarding |
| M4 | Dashboard + hardening |

---

## Milestone plan

| Milestone | Sprints | Epics | Exit criteria |
|-----------|---------|-------|---------------|
| M1 | S1–S2 | E0, E1, E2, E3 | Login; masters; requirements CRUD+SLA |
| M2 | S3–S4 | E4 | Candidates + duplicates + select |
| M3 | S5–S6 | E5, E6 | Offer → join path works E2E |
| M4 | S7–S8 | E7, E8, E9 | Dashboard parity; import; CI; metrics |

## Sprint ceremonies (lightweight)

- Planning: pick stories by dependency graph  
- Daily: blockers  
- Review: demo journey slice  
- Retro: doc gaps  

## Capacity note

Solo developer: prefer vertical slices each sprint (API+UI for one story).
